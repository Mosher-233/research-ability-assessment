# 研究能力评价系统用户手册

## 1. 系统概述

研究能力评价系统是一个基于AI Agent的大学生研究能力评价平台，旨在帮助教师对学生的研究能力进行全面、客观的评估。系统通过收集学生的研究证据，利用AI技术进行能力推理，并生成详细的评价报告。

### 1.1 系统功能

- **用户管理**：支持教师和学生用户的注册、登录和身份验证
- **任务管理**：教师可以创建研究任务并分配给学生
- **证据管理**：学生可以提交研究证据，教师可以查看和管理证据
- **能力评估**：AI Agent自动分析证据，评估学生的研究能力维度
- **结果管理**：查看和管理评估结果
- **报告生成**：生成详细的研究能力评价报告

### 1.2 系统架构

系统采用前后端分离架构：
- **后端**：Go语言 + Gin框架 + PostgreSQL + Neo4j
- **前端**：Vue 3 + TypeScript + Element Plus

## 2. 快速开始

### 2.1 环境要求

- **后端**：Go 1.20+, PostgreSQL 14+, Neo4j 5.0+
- **前端**：Node.js 16+, npm 8+

### 2.2 系统部署

#### 2.2.1 后端部署

1. 克隆项目代码
2. 安装依赖：`go mod tidy`
3. 配置数据库连接（修改 `configs/config.dev.yaml`）
4. 启动后端服务：`go run cmd/server/main.go`

#### 2.2.2 前端部署

1. 进入前端目录：`cd frontend`
2. 安装依赖：`npm install`
3. 启动开发服务器：`npm run dev`
4. 构建生产版本：`npm run build`

## 3. 用户操作指南

### 3.1 教师操作

#### 3.1.1 登录系统

1. 打开系统登录页面
2. 输入邮箱和密码
3. 点击「登录」按钮

#### 3.1.2 创建任务

1. 登录后进入「任务管理」页面
2. 点击「创建任务」按钮
3. 填写任务名称、描述、课程ID、开始日期和结束日期
4. 点击「提交」按钮

#### 3.1.3 分配任务

1. 在任务列表中找到需要分配的任务
2. 点击「分配任务」按钮
3. 选择需要分配的学生
4. 点击「确定」按钮

#### 3.1.4 查看证据

1. 进入「证据管理」页面
2. 选择任务和学生
3. 查看学生提交的证据列表
4. 点击「查看详情」查看证据详细信息

#### 3.1.5 查看评估结果

1. 进入「结果管理」页面
2. 选择任务和学生
3. 查看评估结果列表
4. 点击「查看详情」查看详细评估结果

#### 3.1.6 生成报告

1. 进入「报告管理」页面
2. 点击「生成报告」按钮
3. 选择学生、任务和报告类型
4. 点击「生成报告」按钮
5. 在报告列表中查看和下载生成的报告

### 3.2 学生操作

#### 3.2.1 登录系统

1. 打开系统登录页面
2. 输入邮箱和密码
3. 点击「登录」按钮

#### 3.2.2 查看任务

1. 登录后进入「任务管理」页面
2. 查看分配给自己的任务列表
3. 点击「查看详情」查看任务详细信息

#### 3.2.3 提交证据

1. 进入「证据管理」页面
2. 选择任务
3. 点击「提交证据」按钮
4. 填写证据类型、内容、知识点名称和级别
5. 点击「提交」按钮

#### 3.2.4 查看评估结果

1. 进入「结果管理」页面
2. 查看自己的评估结果列表
3. 点击「查看详情」查看详细评估结果

## 4. 系统功能详解

### 4.1 任务管理

任务管理模块允许教师创建、管理和分配研究任务。每个任务包含名称、描述、课程ID、开始日期和结束日期等信息。教师可以查看任务的学生分配情况和完成进度。

### 4.2 证据管理

证据管理模块允许学生提交研究证据，教师查看和管理证据。证据包括类型、内容、知识点名称和级别等信息。系统支持多种证据类型，如论文、实验报告、项目成果等。

### 4.3 能力评估

系统使用AI Agent对学生的研究能力进行评估，评估维度包括：
- 文献综述能力
- 研究设计能力
- 数据分析能力
- 批判性思维能力
- 学术写作能力

AI Agent会根据学生提交的证据，分析学生在各个维度的表现，并给出相应的评分和评价。

### 4.4 结果管理

结果管理模块允许查看和管理评估结果。评估结果包括总体评分、总体级别、各维度评分和详细推理过程。教师可以查看所有学生的评估结果，学生只能查看自己的评估结果。

### 4.5 报告生成

报告生成模块允许生成详细的研究能力评价报告。报告包括学生的基本信息、任务信息、能力评估结果、详细分析和改进建议等内容。系统支持生成详细报告和摘要报告两种类型。

## 5. 常见问题

### 5.1 登录问题

**问题**：无法登录系统
**解决方案**：
- 检查邮箱和密码是否正确
- 检查网络连接是否正常
- 联系系统管理员重置密码

### 5.2 任务分配问题

**问题**：无法分配任务给学生
**解决方案**：
- 检查学生是否已经注册
- 检查学生ID是否正确
- 联系系统管理员确认权限

### 5.3 证据提交问题

**问题**：无法提交证据
**解决方案**：
- 检查任务是否已经分配
- 检查任务是否在有效期内
- 检查证据内容是否符合要求

### 5.4 评估结果问题

**问题**：评估结果不准确
**解决方案**：
- 提交更多、更全面的证据
- 确保证据内容与任务要求相关
- 联系教师进行人工评估
